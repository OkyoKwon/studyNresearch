# 제품 로드맵

## 제품 비전

- **미션**: 관심 도메인의 지식을 AI와 함께 체계적으로 축적하여, 언제 어디서든 빠르게 참조할 수 있는 개인 지식 허브를 구축한다
- **비전**: 1년 후, PG/MOR/가상자산 규제를 포함한 5개 이상의 도메인 지식이 구조화되어 있고, 새로운 도메인을 30분 이내에 추가할 수 있는 확장 가능한 지식 사이트를 운영한다
- **North Star Metric**: 축적된 도메인 지식 문서 수 (목표: 분기당 30개 이상의 고품질 문서 추가)

---

## 타겟 페르소나

### 페르소나: 지식 탐구자 (본인)

| 항목 | 내용 |
|------|------|
| 역할 | 핀테크/규제 도메인에 관심 있는 개인 학습자 |
| 목표 | 흩어진 도메인 지식을 한 곳에 체계적으로 정리하고 반복 참조 |
| 학습 패턴 | 출퇴근 시 모바일로 빠르게 훑고, 데스크톱에서 깊이 있게 탐색 |
| 핵심 니즈 | 도메인 간 관계 파악, 대표 제품 비교, 규제 변화 추적 |
| 불만 (As-Is) | 노션/블로그에 흩어진 메모, 검색이 어렵고 구조가 없음 |
| 기대 (To-Be) | 도메인별로 정리된 위키, AI가 초안을 작성해 빠른 축적, 모바일에서도 읽기 편함 |

### 핵심 유즈케이스

1. **지식 탐색**: 특정 도메인(예: PG)의 개념, 플로우, 대표 제품을 구조적으로 탐색한다
2. **빠른 참조**: 모바일에서 특정 용어나 제품 정보를 검색하여 즉시 확인한다
3. **지식 축적**: AI가 생성한 콘텐츠를 검토하고 사이트에 추가한다
4. **도메인 확장**: 새로운 관심 도메인을 추가하고 기본 구조를 빠르게 생성한다

## OKR (2026 Q2, 현재 분기)

### Objective 1: 사이트 기반 구축 및 초기 도메인 3개 콘텐츠 확보

- KR 1.1: GitHub Pages에 정적 사이트 배포 완료 (현재: 미배포 --> 목표: 배포 및 접속 가능)
- KR 1.2: PG, MOR, 가상자산 규제 3개 도메인의 기본 문서 구조 완성 (현재: 0개 --> 목표: 3개 도메인, 도메인당 5개 이상 문서)
- KR 1.3: 모바일 반응형 레이아웃 구현 완료 (현재: 없음 --> 목표: 모바일/데스크톱 모두 가독성 확보)

### Objective 2: AI 기반 콘텐츠 생성 워크플로우 확립

- KR 2.1: AI 콘텐츠 생성 프롬프트 템플릿 3종 확보 (개념 설명, 제품 비교, 규제 요약) (현재: 0개 --> 목표: 3종)
- KR 2.2: 도메인별 대표 제품/서비스 카탈로그 페이지 구축 (현재: 0개 --> 목표: 3개 도메인 각 3개 이상 제품)
- KR 2.3: 새 도메인 추가 시 소요 시간 30분 이내 달성 (현재: 측정 불가 --> 목표: 30분 이내)

## PRD 작성자 전달 사항

- **최우선 이니셔티브**: 정적 사이트 프레임워크 설정 + GitHub Pages 배포 (RICE 1순위)
- **핵심 기능 요구**: 마크다운 기반 콘텐츠 관리, 모바일 반응형, 도메인별 네비게이션, 검색
- **OKR 연결**: Q2 Objective 1 (사이트 기반 구축)이 선행되어야 Objective 2 (콘텐츠 워크플로우)가 가능
- **성공 지표**: 3개 도메인, 15개 문서, 모바일 Lighthouse 90+
- **기술 제약**: GitHub Pages 호스팅, 정적 사이트 생성기, 서버 사이드 로직 불가, 인증 불필요
- **추천 기술 스택**: MkDocs Material (파이썬 기반, 마크다운 네이티브, 검색 내장, 반응형 기본 제공) 또는 Docusaurus (React 기반, 확장성 우수)

## 유저스토리 작성자 전달 사항

- **페르소나**: 핀테크/규제 도메인 학습자, 1인 사용, 모바일+데스크톱 병행 사용
- **핵심 유즈케이스 4가지**: 지식 탐색, 빠른 참조, 지식 축적, 도메인 확장 (위 페르소나 섹션 참조)
- **우선순위**: 탐색 > 참조 > 축적 > 확장 순서로 스토리 작성
- **수용 기준 참고**: "30분 이내 도메인 추가", "모바일에서 3탭 이내 원하는 문서 도달" 등 구체적 기준 반영
- **콘텐츠 구조 예시**: 도메인 개요 --> 핵심 개념 --> 비즈니스 플로우 --> 대표 제품 카탈로그 --> 규제/트렌드

## Epic 1: 사이트 인프라 구축

### US-001: MkDocs Material 프로젝트 초기화
- **유저스토리**: 지식 탐구자로서, MkDocs Material 기반의 정적 사이트를 로컬에서 실행하고 싶다, 콘텐츠를 작성하고 미리보기할 수 있는 환경을 확보하기 위해
- **우선순위**: P0
- **스토리포인트**: 3
- **AC (인수 기준)**:
    - [ ] Given MkDocs Material이 설치되지 않은 상태, When `pip install`로 의존성을 설치하고 `mkdocs new`로 프로젝트를 초기화하면, Then `mkdocs serve`로 로컬 서버가 정상 실행되고 기본 페이지가 렌더링된다
    - [ ] Given 프로젝트가 초기화된 상태, When `mkdocs.yml`에 Material 테마 설정을 추가하면, Then Material Design 테마가 적용된 페이지가 표시된다
    - [ ] Given 프로젝트 루트에 `requirements.txt`가 존재하면, When 파일을 확인하면, Then mkdocs-material 버전이 고정(pinning)되어 있다
- **의존성**: 없음
- **기술 노트**: `requirements.txt`에 `mkdocs-material>=9.0` 버전 고정. `mkdocs.yml`에 `theme.name: material` 설정.

### US-002: GitHub Actions 자동 배포 파이프라인
- **유저스토리**: 지식 탐구자로서, main 브랜치에 push하면 사이트가 자동으로 배포되길 원한다, 수동 배포 없이 콘텐츠 업데이트에만 집중하기 위해
- **우선순위**: P0
- **스토리포인트**: 3
- **AC (인수 기준)**:
    - [ ] Given `.github/workflows/deploy.yml` 파일이 존재하는 상태, When main 브랜치에 push하면, Then GitHub Actions가 자동으로 트리거되어 빌드 및 배포가 실행된다
    - [ ] Given 빌드가 성공한 상태, When GitHub Pages URL에 접속하면, Then HTTPS로 사이트가 정상 표시된다
    - [ ] Given 빌드가 성공한 상태, When push 후 5분 이내에 사이트에 접속하면, Then 최신 콘텐츠가 반영되어 있다
    - [ ] Given 마크다운에 깨진 링크가 포함된 상태, When GitHub Actions 빌드가 실행되면, Then `mkdocs build --strict` 옵션으로 빌드가 실패하고 오류를 보고한다
- **의존성**: US-001 완료 후
- **기술 노트**: `mkdocs gh-deploy` 또는 `actions/deploy-pages` 활용. `--strict` 옵션으로 링크 체크 포함.

### US-008: 모바일 반응형 레이아웃 검증
- **유저스토리**: 지식 탐구자로서, 출퇴근 중 모바일에서도 문서를 편하게 읽고 싶다, 시간과 장소에 구애받지 않고 학습하기 위해

    - [ ] Given 모바일에서 사이트에 접속한 상태, When 네비게이션 메뉴를 열면, Then 햄버거 메뉴가 정상 동작하고 도메인별 섹션 이동이 가능하다

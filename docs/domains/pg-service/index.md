# PG (Payment Gateway) 개요

**PG(Payment Gateway)** 란 온라인 가맹점과 금융기관(카드사, 은행 등) 사이에서 결제 데이터를 안전하게 중계하고 처리하는 전자결제 대행 서비스다. 가맹점이 개별 금융기관과 직접 계약하지 않고도 다양한 결제수단을 통합 제공할 수 있게 해준다.

---

## 왜 알아야 하는가

- **커머스/핀테크 필수 인프라**: 온라인 비즈니스를 운영하려면 결제 시스템 이해가 필수다.
- **서비스 설계에 직접 영향**: 결제 플로우, 정산 주기, 수수료 구조가 비즈니스 모델과 사용자 경험을 결정한다.
- **규제·보안 이슈**: PCI DSS, 전자금융거래법 등 컴플라이언스를 이해해야 안전한 서비스를 구축할 수 있다.
- **글로벌 확장**: 해외 결제를 지원하려면 PG와 [MOR(Merchant of Record)](../mor-service/index.md)의 차이를 이해해야 한다.

---

## 핵심 키워드

| 키워드 | 설명 |
|--------|------|
| **매입(Acquiring)** | 카드사가 가맹점의 매출 전표를 접수하고 대금을 지급하는 과정 |
| **발급(Issuing)** | 카드사가 소비자에게 카드를 발급하는 역할 |
| **정산(Settlement)** | PG가 가맹점에 수수료를 차감한 뒤 대금을 지급하는 과정 |
| **VAN** | 카드 승인 전문을 카드사에 중계하는 부가가치통신망 |
| **간편결제** | 카드 정보를 사전 등록하고 비밀번호·생체인증만으로 결제 |
| **빌링키** | 정기결제를 위해 PG가 발급하는 카드 대체 토큰 |
| **PCI DSS** | 카드 데이터 보안의 국제 표준 |
| **3D Secure** | 카드 온라인 결제 시 본인인증 프로토콜 |

## 관련 도메인

- [MOR (Merchant of Record)](../mor-service/index.md) — PG와 달리 **판매자(Merchant of Record)** 역할까지 대행하는 모델. 세금, 환불, 컴플라이언스를 MOR 사업자가 책임진다. PG는 결제 중계만 담당하므로 가맹점이 직접 세금·환불을 처리해야 한다.
